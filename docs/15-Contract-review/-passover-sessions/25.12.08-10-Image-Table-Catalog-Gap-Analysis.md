## Architecture Notes

Current store has two layers:
1. **Calculator Section (CALC):** Input fields
2. **Approach Sections (INCOME/SALES):** Narrative analysis

**Gap:** Missing third layer for **display table fields** that format calculator data for report rendering.

**Recommendation:** Add subsections:
- `income-expense-table` for display fields
- `income-direct-cap-table` for display fields
- `sales-comparison-grid` for all comp data
- `rental-survey` as new section
