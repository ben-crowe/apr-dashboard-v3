   - APR Dashboard V3 (job management + test form)

**ISSUE #1: Property Type Not Displaying in Dashboard**

- Impact: Data exists in database, just doesn't display
- Workaround: Manually select property type again
- Fix: Update component to read `job.property_type` or add camelCase mapping

**ISSUE #2: Email Notification Not Sent**
- Severity: HIGH (critical for workflow)

- Code exists: `send-appraisal-request` edge function configured correctly
- Possible causes:
  1. Function not deployed to Valta Website's Supabase project
  2. Resend API key issue
  3. Silent failure in function
- Action needed: Check Supabase Edge Function logs, verify deployment

**ISSUE #3: Property Type Multi-Select Not Available on Client Form**
- Severity: Low (client can only pick one, appraiser adds more later)
- Problem: Client form has single dropdown, dashboard has checkboxes
- Impact: Client can only submit one property type

## Testing Protocol - Important Rules

### DO:
- Use Playwright for browser automation and console monitoring
- Query Supabase directly to verify data
- Compare dashboard vs Valcre field-by-field
- Take screenshots of all errors
- Report specific file paths and line numbers

### DON'T:
- Create multiple test jobs in Valcre (client sees live system)
- Use fake/random field names (use realistic test data)
- Make assumptions without data verification
- Fix issues (just document them)
