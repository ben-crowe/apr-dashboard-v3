# Data Flow Summary - APR Dashboard to Report Generation

**Created:** 2025-12-15
**Purpose:** Comprehensive mapping of data flow from user inputs through calculator engine to final report

---

## Executive Summary

This document synthesizes findings from field extraction, input/output classification, and destination mapping to provide a complete picture of how data flows through the APR Dashboard system.

**Key Findings:**
- **519 total fields** in fieldRegistry.ts
- **169 unique fields** in Word HTML report template
- **377 user input fields** manually entered in TDD Dashboard
- **59 calculated fields** in registry, **15 matched to Valcre calculator outputs**
- **38 fields reused** across multiple report sections
- **10 report sections** with varying field density (0-78 fields per section)

## Field Name Convention Mapping

### Naming Patterns

| System | Convention | Example |
|--------|------------|---------|
| **fieldRegistry.ts** | `kebab-case` | `client-company`, `calc-pgi` |
| **Word HTML** | `PascalCase_Underscore` | `Client_Company`, `Subject_PropertyName` |
| **Valcre Workbook** | `PascalCase_Underscore` + Prefix | `IA_DirCap_PGI`, `L_Ceilings` |

### Prefix Meanings

| Prefix | System | Meaning |
|--------|--------|---------|
| `intake-*` | Registry | Client intake form fields (V3 Dashboard) |
| `loe-*` | Registry | Letter of Engagement / Quote fields |
| `img-*` | Registry | Image upload slots |
| `calc-*` | Registry | Calculated fields in dashboard |
| `survey1-5-*` | Registry | Rental survey comparable data |
| `comp1-3-*` | Registry | Sales comparable data |
| `IA_*` | Valcre | Income Analysis calculations |
| `L_*` | Valcre | List/lookup reference tables |
| `Subject_*` | Word HTML | Property/subject details |
| `Client_*` | Word HTML | Client information |
| `Report_*` | Word HTML | Report metadata |

## Critical Conflicts & Gaps

### Input/Output Conflicts (7 fields)

Fields where Registry says "user-input" but Valcre marks as "calculated":

1. `ceilings` → `L_Ceilings` (🔢 Calculated)
2. `client-address` → `Client_Address` (🔢 Calculated)
3. `client-company` → `Client_Company` (🔢 Calculated)
4. `doors-windows` → `L_DoorsWindows` (🔢 Calculated)
5. `electrical` → `L_Electrical` (🔢 Calculated)
6. `elevator` → `L_Elevator` (🔢 Calculated)
7. `foundation` → `L_Foundation` (🔢 Calculated)

**Recommendation:** Verify if these should be user inputs or lookups. The `L_*` prefix suggests they're lookup lists (dropdowns) rather than calculations.

### Unmapped Dashboard Fields (492 fields)

**Dashboard-Specific Fields (not in Valcre):**
- Client Intake: `intake-*` fields (22 fields)
- LOE/Quote: `loe-*` fields (13 fields)
- Image Management: `img-*` fields (100+ fields with captions)

**Likely Naming Mismatches:**
- Rental Surveys: `survey1-*` through `survey5-*` (150+ fields)
- Sales Comps: `comp1-*` through `comp3-*` (24 fields)
- Calculator inputs: `calc-*` fields (many unmapped)

**Action Required:** Manual review to determine which unmapped fields should:
1. Be added to Valcre workbook (new dashboard-specific fields)
2. Be renamed to match Valcre convention
3. Remain dashboard-only (intake, LOE, images)
