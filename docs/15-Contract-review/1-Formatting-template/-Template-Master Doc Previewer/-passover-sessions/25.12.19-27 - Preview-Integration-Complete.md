## 🔑 Solution Implemented

### 1. Store Template System ✅

**Added to reportBuilderStore.ts:**

```typescript
// State
previewTemplate: "" // Cached PREVIEW-Master.html

// Load template once (with caching)
loadPreviewTemplate: async () => {
  if (currentTemplate) return currentTemplate; // Cache hit
  const response = await fetch('/PREVIEW-Master.html');
  const html = await response.text();
  set({ previewTemplate: html });
  return html;
}

// Interpolate fields (synchronous - avoids stale state)
interpolateTemplate: (sections, template) => {
  let html = template;

  // Build field map from sections + subsections
  const fieldMap = new Map<string, string>();
  sections.forEach(section => {
    section.fields?.forEach(field => {
      fieldMap.set(field.id, formatFieldValue(field));
    });
    section.subsections?.forEach(subsection => {
      subsection.fields?.forEach(field => {
        fieldMap.set(field.id, formatFieldValue(field));
      });
    });
  });

  // Replace {{field-id}} with actual values
  fieldMap.forEach((value, fieldId) => {
    html = html.replaceAll(`{{${fieldId}}}`, value);
  });

  return html; // Unreplaced placeholders left for toggle
}

// Format values by type
formatFieldValue: (field) => {
  if (!field.value && field.value !== 0) return '';

  switch (field.type) {
    case 'currency':
      return `$${Number(field.value).toLocaleString('en-US', {
        minimumFractionDigits: 2,
        maximumFractionDigits: 2
      })}`;
    case 'percentage':
      return `${field.value}%`;
    case 'date':
      return new Date(field.value).toISOString().split('T')[0];
    case 'image':
      return Array.isArray(field.value) ? field.value[0] || '' : String(field.value);
    default:
      return String(field.value);
  }
}
```

## 🐛 Problems Solved

### Problem 1: Async Stale State Pattern

**Symptom:** Initial implementation had async interpolateTemplate that loaded template inside:
```typescript
// BAD PATTERN:
interpolateTemplate: async (sections) => {
  const template = await get().loadPreviewTemplate();
  // ... interpolation
  return html;
}
```

**Issue:** Calling `get()` after `await` in Zustand can have stale state issues.

**Solution:** Made interpolation synchronous, load template separately:
```typescript
// GOOD PATTERN:
interpolateTemplate: (sections, template) => {
  // Synchronous - no await, no stale state
  // ... interpolation
  return html;
}

// Caller loads first:
const template = await get().loadPreviewTemplate(); // Cached
const html = get().interpolateTemplate(sections, template);
```

**Result:** Template loads once per operation, interpolation is fast and safe.

---

### Problem 2: Multi-Level Field Structure

**Challenge:** Fields exist at two levels:
- `section.fields[]` - Top-level fields
- `section.subsections[].fields[]` - Nested fields

**Solution:** Iterate both levels:
```typescript
sections.forEach(section => {
  section.fields?.forEach(field => {
    fieldMap.set(field.id, formatFieldValue(field));
  });
  section.subsections?.forEach(subsection => {
    subsection.fields?.forEach(field => {
      fieldMap.set(field.id, formatFieldValue(field));
    });
  });
});
```

**Result:** All 889 fields accessible for interpolation.

## 📊 Technical Implementation Details

### Data Flow (TDD → Preview)

```
TDD Dashboard
  ↓ (user edits field)
  updateFieldValue()
  ↓
Zustand Store (sections updated)
  ↓ (automatically triggers)
  generatePreview()
  ↓
  loadPreviewTemplate() → fetch /PREVIEW-Master.html (cached)
  ↓
  interpolateTemplate(sections, template)
  ↓
  Replace {{field-id}} → actual values
  ↓
  set({ previewHtml: html })
  ↓
PreviewPanel (reads previewHtml)
  ↓
PreviewRenderer (iframe with zoom/pan)
  ↓
User sees updated preview
```

### Caching Strategy

**Template Loading:**
```typescript
loadPreviewTemplate: async () => {
  const currentTemplate = get().previewTemplate;
  if (currentTemplate) {
    return currentTemplate; // Cache hit - instant return
  }

  // Cache miss - fetch and store
  const response = await fetch('/PREVIEW-Master.html');
  const html = await response.text();
  set({ previewTemplate: html });
  return html;
}
```

**Performance:**
- First call: ~50-100ms (fetch from /public/)
- Subsequent calls: <1ms (cache hit)
- Template persists in store for entire session

## 🎓 Key Learnings

### 1. Zustand Async Pattern Best Practice

**Avoid:**
```typescript
action: async () => {
  const data = await fetch();
  const state = get(); // ⚠️ Stale state risk
  set({ combined: state + data });
}
```

**Prefer:**
```typescript
action: async () => {
  const data = await fetch();
  set({ data }); // ✅ Set immediately
}

syncAction: () => {
  const state = get(); // ✅ No await before this
  const data = get().data;
  set({ combined: state + data });
}
```

**Lesson:** Minimize `get()` calls after `await` in async actions.

---

### 2. Template vs. Generator Pattern

**Generator Pattern (Old):**
- ❌ HTML embedded in TypeScript
- ❌ Hard to visualize final output
- ❌ Changes require code edits
- ❌ No WYSIWYG editing

**Template Pattern (New):**
- ✅ HTML in separate file
- ✅ Edit directly in Claude Desktop or browser
- ✅ Visual fidelity to final output
- ✅ Toggle field IDs built-in

**When to use each:**
- Template: Known structure, many instances (reports, emails)
- Generator: Dynamic structure, complex logic

---

### 3. Field Interpolation Strategies

**Option A: Regex replacement** (chosen)
```typescript
html = html.replaceAll(`{{${fieldId}}}`, value);
```
- ✅ Simple, fast
- ✅ Works with any placeholder format
- ⚠️ Can't validate field existence

**Option B: Template engine** (overkill for this)
```typescript
import Handlebars from 'handlebars';
const template = Handlebars.compile(html);
const result = template(data);
```
- ✅ Rich features (loops, conditionals)
- ❌ Dependency overhead
- ❌ Requires different placeholder syntax

**Decision:** Regex replacement sufficient for simple field replacement.
