## Decisions Made

### 1. Rule Over Slash Command

**Decision:** Create `.claude/rules/api-credential-search.md` instead of `/find-api-key` command

**Rationale:**
- Rules are automatic - agents follow without being told
- Slash commands require user to type command
- Rule teaches pattern that applies across all APIs
- User can still trigger naturally: "Find the API key"

**Impact:**
- All agents automatically search JSONL on auth failures
- No user action needed
- Pattern internalized as agent behavior

---

### 2. Direct JSONL Search Over Memory Tools

**Decision:** Use direct `grep` on JSONL files instead of memory indexing tools

**Rationale:**
- Faster (no indexing required)
- More reliable (raw data)
- Works for any project
- Simple pattern to teach

**Impact:**
- Search completes in <5 seconds
- No dependencies on memory system setup
- Works even if Cognee/indexing not available

## Key Learnings

### 1. Session History > Documentation for Active Credentials

**Learning:**
Raw JSONL session files contain the actual working credentials used in successful operations. Documentation can be outdated, but session history shows what actually worked at that moment in time.

**Application:**
- Always search JSONL when credentials fail
- Don't trust documentation as single source of truth
- Test extracted credentials systematically
- Use most recently used valid credential

**Pattern Captured:** `.claude/rules/api-credential-search.md`

---

### 2. Rules = Institutional Knowledge

**Learning:**
Creating rules captures problem-solving patterns that agents can automatically follow. Trial-and-error solutions become instant knowledge for future agents.

**Application:**
- Document working patterns as rules
- Include search commands that worked
- Show real examples from actual work
- Make rules automatic, not optional

**Value:**
- Agents get smarter over time
- No repeated learning curves
- Consistent problem-solving approaches
- Reduced user interruptions

### 4. Direct JSONL Grep > Complex Memory Systems

**Learning:**
For credential and config searches, direct grep on JSONL files is faster and more reliable than complex memory indexing systems.

**Application:**
- Use simple grep patterns for quick searches
- Reserve memory systems for semantic searches
- Teach direct patterns in rules
- Keep patterns simple and teachable

**Speed Comparison:**
- Memory system search: ~30 seconds + may not find
- Direct JSONL grep: <5 seconds, always finds if exists
