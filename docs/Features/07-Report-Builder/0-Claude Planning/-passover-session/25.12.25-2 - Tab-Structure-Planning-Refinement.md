## 📊 Key Discoveries

### 1. System Architecture: Two Distinct Roles

**TDD Dashboard = Central Data Capture Hub (Per Job)**
- Receives data from MULTIPLE sources: V3, S1, S2, S3, APIs, manual input
- Consolidates ALL appraisal data for a single job
- Single source of truth for that job's field values
- User role: Data entry, consolidation, quality control

**Report Builder = Output Generation Tool**
- Reads data FROM TDD Dashboard only
- Generates multiple report types FROM the same data
- Current: Appraisal Report
- Future: Financial Reports, XYZ Reports, other variants
- User role: Report customization and export

**Critical Insight:** Data flows ONE direction: TDD → Report Builder. This enables multiple report types from single validated data source.

### 2. Field Management Hierarchy (Critical Rules)

**S1 - CLIENT INTAKE (V3)**
- Client name, company, property address, loan info
- All fields fully editable in S1
- No internal "Managed Elsewhere" links
- Appears as read-only + link in numbered tabs (e.g., Tab 01)

**S2 - LOE PREP (V3)**
- Appraiser name, credentials, job details, financial terms
- All fields fully editable in S2
- No internal "Managed Elsewhere" links
- Populates to Tab 01 (COVER - appraiser info) and Tab 22 (CERTIFICATION - signature/credentials)
- Appears as read-only + link in both tabs

**S3 - IMAGE MANAGEMENT**
- Consolidates all images organized by destination
- S3-01 (Cover), S3-03 (Maps), S3-07 (Property photos), S3-08 (Subject), S3-09 (Sales comp photos), S3-10 (Sales comp maps), S3-11 (Rental photos)
- All images fully editable in S3
- Appears as image + link in destination tabs (read-only)

**Core Rule:** Fields managed in S1/S2/S3 are NEVER editable in numbered tabs - always read-only with "Managed in [X]" link.

### 3. Critical UI Pattern Discovery

**WRONG Pattern (Current Issue):**
```
Client Name: [🔗 Managed in S1]
             ← User can't see the value (looks empty/broken)
```

**CORRECT Pattern (Required):**
```
Client Name: John Smith [🔗 Edit in S1]
             ^^^^^^^^^^^  ^^^^^^^^^^^^^
             Value visible  Link to source
```

**Why this matters:** User needs to see the actual data while reviewing the rendered report (previewer), not just a link. Provides context and reassures them the data exists.

### 4. Tab Structure Rationalization

**Problems Identified:**
- Tab 16 (LAND VALUE): Only 1 field → redundant, belongs in Cost Approach
- Tabs 19-20 (RENT COMPS/ROLL): Income data scattered from income calculation → should consolidate
- Tab 15 (VALUATIONS): Originally consolidated all three approaches, preventing separate access
- Missing tabs (03, 04, 07, 17): Gaps created confusion in sequential flow

**Solutions Applied:**
- Tab 16 consolidated into Tab 17 (B-COST APPROACH)
- Tabs 19-20 consolidated into Tab 15 (A-INCOME APPROACH)
- Tab 15 renamed to A-INCOME APPROACH (separate from cost/sales comp)
- Tab 17 renamed to B-COST APPROACH
- Tab 18 renamed to C-SALES COMPS APPROACH
- Missing tabs (03, 04, 07) remain noted as consolidated but gaps acknowledged

### 5. Three Appraisal Approaches Now Clearly Separated

```
├── 15 - A-INCOME APPROACH
│   └── Includes: Rental income, rents from Tab 19-20, NOI, capitalization
│
├── 17 - B-COST APPROACH
│   └── Includes: Land value (from Tab 16), building cost, depreciation
│
└── 18 - C-SALES COMPS APPROACH
    └── Includes: Comparable sales, adjustments, indicated value
```

Each approach now has:
- Own numbered tab
- Related data consolidated within it
- Clear labeling (A, B, C) showing these are the three approaches
- Sequential flow from analysis → approaches → reconciliation → certification

### 6. Editor Panel Must Mirror TDD Dashboard

**CURRENT STATE (Wrong):**
- Missing S1, S2, S3 tabs
- Short/abbreviated tab names without numbers
- Doesn't match TDD structure
- User confusion when switching between tools

**DESIRED STATE (Correct):**
- S1 - CLIENT INTAKE (V3)
- S2 - LOE PREP (V3)
- S3 - IMAGE MANAGEMENT (with subsections)
- 01-22 numbered tabs with full names
- Identical structure to TDD Dashboard
- User navigates same way in both tools

**Why this works:** As user views page 15 in report previewer, they see Tab 15 in editor. Tab contains income approach fields (read-only with links to S3 images). Contextual continuity maintained.

## 🔑 Core Principles Established

1. **Per Job Scoping:** Single source of truth applies per appraisal, not globally
2. **Field ID Permanence:** Registry field IDs never change - never override with V3/Valcre names
3. **Consolidation Logic:** Keep data together where it logically belongs
4. **UI Visibility:** Show actual values with links, never link-only
5. **Tab Flow:** Tabs represent sequential flow of appraisal report
6. **Editor Panel Mirroring:** Editor tabs must match TDD tabs exactly for user consistency
7. **Managed Elsewhere Pattern:** Read-only + link in numbered tabs, fully editable in S-tabs
