**What We Discovered:**
- **Correct skill structure:** Folder with SKILL.md (caps required) + optional reference docs
- **YAML frontmatter pattern:** name, description, category, activation, priority, version, date
- **Skills can reference commands:** Best practice is single source of truth - keep docs in commands, skills trigger them
- **Terminal links impossible:** Tested multiple formats (file://, zed://, OSC 8 escape sequences) - none work in Claude Code/Zed terminal
- **Skills are command triggers:** Most valuable use is just triggering slash commands, not duplicating documentation
- **9 skills correct, 4 need fixing:** Audited all user's skills, identified broken ones

**What Changed:**
- **apr-report-builder-domain:** Created proper SKILL.md that references /check-apr-domain-knowledge command
- **open-file-in-zed:** Fixed to only open files when user explicitly asks (removed proactive opening)
- **Skill understanding:** Clarified that SKILL.md must be all caps, can reference external files

**Decisions Made:**
- **Decision:** Use `open -a Zed <filepath>` for file opening instead of clickable links
  - **Why:** Terminal links don't work in this environment after extensive testing
  - **Impact:** When user asks "open [file]", agent runs command and provides line ranges

- **Decision:** Skills should trigger commands, not duplicate documentation
  - **Why:** Single source of truth - update one file (command), skill just references it
  - **Impact:** APR skill SKILL.md is ~30 lines, references 1142-line command doc

- **Decision:** Open files ONLY when user explicitly requests
  - **Why:** User doesn't want automatic file opening after every creation
  - **Impact:** Agent waits for "open [file]" trigger, doesn't be proactive
