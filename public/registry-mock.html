My layout notes

  1. Three name columns during reconciliation: our dashboard field (V3/V4) first, then Valta (the client's requested names), then Valcre. Sitting ours beside Valta shows which of our names to change. Amber = mismatch to fix (Company Name → Client Company Name, Valcre Job ID → Job Number, Appraisal Status → Job Status, Property Rights Appraised → Interest Appraised). Once aligned, ours = Valta and the first column collapses away.
  2. One title, "Master Registry (Valta)." Replaces the two confusing tabs. It reads as one registry — the client's Valta master list — in Valta's own naming, not our improved V4 ids.
  3. Views are filter chips, not tabs. Dashboard / V3 / V4 / Intake / LOE become multi-select checkboxes that subset the same rows. Every field still lives in the one registry; the chips just organize what you see. Multi-select so you can view V3 + V4 together.
  4. Valta Field = Label on top, id underneath. Two only — the redundant third "Field Name" is gone. The id is Valta's own (PascalCase, e.g. ClientFirstName).
  5. Valcre sits immediately beside Valta (the divider line), so the two ids are side by side for quick eyeballing — not pushed off to the far right. Same shape: Label on top, Valcre id (dotted) under.
  6. Maps column — ✓ when the field has a Valcre target, "partial" for composites, "—" when it has no Valcre field at all (report-only fields). Blank Valcre cell = doesn't map.
  7. Dropdowns are the ugly ones. Their Valcre id is a bare custom-field code (CF-number) with no readable name — flagged in amber so it's obvious those are the ones stuck with Valcre's custom-field scheme.
  8. Endgame — our kebab id becomes the standard. Once names align, the pitch is the client adopts our kebab convention as the master id (he'd defer to the better one anyway). Then there are two ids per field by design: the client's id folds into ours, and ours maps outward to wherever it needs to go — Valcre, the LOE template, e-sign, etc. Ours proves the more versatile, single id that every downstream system maps from.
