## 📋 Session Overview

This session focused on establishing proper file organization and page numbering alignment for the APR report builder system. The primary goal was to ensure that image files (screenshots), single HTML files, and batch HTML files all reference the same page numbers consistently, using the screenshot images as the source of truth.

---

## 🎯 Topics Identified

### File Organization Structure
- **Three file types:** Images (Page-XX.png), Single HTML (Page-XX.html), Batch HTML (Page-XX-XX.html)
- **Front matter pages:** Cover, Conditions, Letter, Table of Contents (no page numbers)
- **Numbered pages:** Start at Page 1 (not PDF page numbers)
- **Location structure:**
  - `doc-page-images/` - Screenshot references from PDF
  - `doc-page-html/doc-pages-html-single/` - Individual HTML pages
  - `doc-page-html/doc-pages-html-formatted/` - Formatted batch files

### Page Numbering Alignment
- Screenshot images show actual document page numbers at bottom
- HTML files must match image page numbers exactly
- Front matter pages (a, b, c, d) have NO page numbers
- Numbered pages start at 1 after front matter

### Screenshot Images as Source of Truth
- Images captured from original PDF show true page numbers
- Client does not number cover, conditions, letter, or TOC pages
- Numbering starts where images show it (Page 1)
- All file renaming based on what appears in screenshot footers

---

## ✅ Decisions Made

1. **Images are source of truth** - Page numbers from PDF screenshots determine all naming
2. **HTML must match images** - File names and internal content synchronized
3. **Front matter no numbers** - Cover, conditions, letter, TOC use descriptive names only
4. **Batch files named by content** - Check actual footer page numbers, not assumptions
5. **Sequential numbering** - Pages 1-74 sequential, offset from original PDF page numbers

## 🔄 Cross-Reference Verification Process

Established three-way verification:

1. **Image shows page X** at bottom →
2. **HTML file Page-X.html** contains page X content →
3. **Batch file** footer shows page X

**Verification method:**
- Check image bottom for page number
- Check HTML `<title>` tag for page number
- Check batch HTML footer `<span class="page-num">` for page number
- All three must match

## 📝 Technical Details

### Python Script - Update HTML Page Numbers
```python
import re
import os

files = sorted([f for f in os.listdir('.') if f.startswith('Page-') and f.endswith('.html')])

for filename in files:
    page_num = int(filename.replace('Page-', '').replace('.html', ''))

    with open(filename, 'r', encoding='utf-8') as f:
        content = f.read()

    content = re.sub(r'<title>.*?Page \d+.*?</title>',
                     f'<title>Appraisal Report - Page {page_num}</title>',
                     content, flags=re.IGNORECASE)

    with open(filename, 'w', encoding='utf-8') as f:
        f.write(content)
```

### File Naming Conventions
- **Front matter:** `-[letter]-[description].html` (e.g., `-a-cover.html`)
- **Single pages:** `Page-[NN].html` (two-digit padding)
- **Batch files:** `Page-[NN]-[NN].html` (page range)
- **Special batch:** `Page-01-02 & B-D.html` (combined front matter + numbered)
