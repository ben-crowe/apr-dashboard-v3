APR Field Registry — Capture Methods Board

9-panel reference of headless screenshot capture techniques. FIXED-width variant — panels stay 400px regardless of viewer size. Whitespace fills extra space.

#1

Headless capture: field-registry-v6

Source: https://apr-dashboard-v3.vercel.app/field-registry-v6.html (headless agent-browser open + screenshot --annotate, NOT km-shot). Page title: 'Valta Field Registry Explorer'. DOM snapshot: 552 ARIA elements — 17 buttons, 3 headings (Valta Field Registry Explorer / Columns / Rule Audit), 8 form inputs (textbox/combobox/checkbox), 0 links. Two modal panels detected via close-button (✕) heuristic: 'Columns' panel @e7 + 'Rule Audit' panel @e9. Screenshot dims: 1280×577 px (viewport-only — page is much taller, full body scroll content not captured by default; pass --full for full scrollable). Annotate flag drew 552 red boxes mapping every clickable + landmark element to @eN refs. Console: clean (no errors). Workflow: agent-browser open URL → snapshot -i (DOM with refs) → screenshot --annotate (visual + refs) → console (errors) → close.

#2

Headless capture: clean (no annotate)

Source: same URL, agent-browser open + screenshot (no flags). Pure visual — no red boxes, no element refs. Screenshot dims: 1280x577. Use case: client preview, design review, before/after compare. Audience: designer + client. Workflow: agent-browser open URL → screenshot <path> → close. Note: viewport-only by default; pass --full for entire scrollable page.

#3

Headless capture: zoom region

Source: PIL-crop on the clean screenshot. Crop bounds (640x288 logical viewport): x=140-1120, y=130-530 → 980x400 cropped pixels. Use case: highlight a specific section (here: FIELD REGISTRY data grid) without distracting top chrome. Workflow: agent-browser open URL → screenshot full → PIL crop (left,top,right,bottom) → save. Alternative: agent-browser snapshot to find element ref, get bounding rect via eval, then PIL-crop using those coords (precise pinpoint).

#4

Headless capture: full-screen app size

Source: same URL. agent-browser open URL → set viewport 1920 1080 → screenshot → close. Screenshot dims: 1920x1080. Native desktop viewport. Sees the whole field-registry table with many more rows than the default 1280x577 viewport. Note: set viewport is SAFE in standalone agent-browser (its own isolated headless Chrome). BANNED against --cdp 9222 (would resize the live KM-Exp app). Use case: realistic full-page client preview at full HD, marketing demos, what-the-app-looks-like-on-a-laptop captures.

#5

Headless capture: full page (--full)

Source: agent-browser open URL → screenshot --full → close. Captures the ENTIRE scrollable document, not just the viewport. Native dims: 1265x1906 — page is ~3.3x taller than the default 577px viewport. Shows all 45 Valta Dashboard fields from JobNumber to the bottom of the table in one image. Use case: full page reviews for clients ('here's the whole page top to bottom'), long landing pages, scrollable dashboards, multi-fold marketing pages where the fold-only capture is misleading. Trade-off: very tall image — display at 1200 wide → 1808 tall on canvas. Use this when content height matters more than visual compactness.

#6

Multi-step: click tab → capture section

Delegation: 'open URL, click Logic Fields tab, capture top half of viewport'. Workflow (no coordinate guessing): (1) agent-browser open URL. (2) snapshot -i → DOM tree with refs. Found 3 matches for 'Logic Fields' — picked @e5 (button, the tab), ignored @e15 (filter label) + @e26 (filter checkbox) by role. (3) agent-browser click @e5 → click by REF, not pixel coords; browser handles the mapping. (4) snapshot -i again to confirm new DOM state — tab is now active, content swapped to Cascade Logic with Scenario Groups + Group 1 Property Rights triggers. (5) screenshot /tmp/...png — 1280x577 viewport. (6) PIL-crop top half → 1280x288. (7) close. Key principle: read DOM → identify by role+text → click ref → re-snapshot to verify → capture. Never guess coords. Never click without verifying the AFTER state.

#7

Size: Section (1280×577)

Section size = default viewport, 1280×577. Above-the-fold view. Same dims as Card 2 (clean viewport on Field Registry tab). Captured AFTER clicking Logic Fields tab — shows the Cascade Logic content. Cmd: agent-browser screenshot <path> (no flags). Use when: 'show me the section', 'above the fold', 'first screen view'.

#8

Size: Monitor (1920×1080)

Monitor size = full HD desktop viewport. Cmd: agent-browser set viewport 1920 1080 → screenshot. Sees significantly more content than Section since the viewport is bigger — Group 3 'No logic configured' + the '+ Create Logic Field' button + the RULE EXPLORER section at the bottom are all visible now. Use when: 'monitor size', 'desktop preview', 'show me what a laptop user sees'. Reminder: set viewport is safe on standalone agent-browser. BANNED against --cdp 9222.

#9

Size: Full (entire scrollable doc)

Full size = entire scrollable document, regardless of viewport. Cmd: agent-browser screenshot --full <path>. Edge case observed here: --full returned 1920×1080 — SAME as Monitor — because the Logic Fields page CONTENT fits within the 1920×1080 viewport (no scroll required). So Full == Monitor for short pages, but Full >> Monitor on tall pages like the Field Registry tab (Card 5 = 1265×1906). Use when: 'full page', 'top to bottom', 'all of it'. Don't assume Full is always bigger than Monitor — depends on page content height.
