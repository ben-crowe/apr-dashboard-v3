### B. Agent/Infrastructure Troubleshooting & Artifact Creation

A significant portion of the session was dedicated to resolving systemic issues in agent behavior and configuration, leading to the creation of essential new reference artifacts.

#### 1\. Playwright MCP Configuration Fix

  * **Root Cause:** Agents were searching memory/old docs instead of checking the live configuration file (`~/.config/claude/config.json`), leading to confusion (e.g., chasing Docker/WSL/wrong product configs). The actual fix was the simple command: `claude mcp add playwright -- npx -y @playwright/mcp@latest`.
  * **Solution Artifacts:**
      * **`/mcp-status` Diagnostic Script:** Created to instantly display the active Playwright configuration, eliminating the need for manual debugging.
      * **CLAUDE.md Update:** Updated to point to this diagnostic script and emphasize that agents must **check the config file first**.

#### 2\. Supabase Deployment Process Fix

  * **Root Cause:** Agents continually referenced old, conflicting, or slightly incorrect deployment documentation (`DEPLOYMENT.md`, `pmem` search), leading to failed attempts and flag errors (e.g., using `--project-ref` instead of the correct, simplified process).
  * **Solution Artifact:**
      * **`.claude/COMMANDS.md`:** Created as the **Single Source of Truth** for all core commands (Deployment, MCP, Sessions), explicitly instructing agents to **ignore** all other conflicting documents and memory entries.

#### 3\. Agent Delegation & Token Management

  * **Issue:** Marcel attempted to resolve database-level errors himself instead of immediately routing to the `backend-developer` agent, causing delays. Additionally, full Playwright MCP output (approx. 10.5k tokens) was filling context quickly.
  * **Resolution:** New persona rules established for Marcel:
      * **Immediate Handoff:** Route all **database, schema, and migration errors** immediately to `backend-developer` agent.
      * **Output Summarization:** After running Playwright, **extract only key findings** and drop verbose HTML/DOM snapshots to conserve context tokens.

### C. Live Testing & Field Mapping Gaps

Live testing after the initial fixes revealed several critical field mapping gaps, establishing the next set of development tasks.

#### Critical Mapping Bugs (Valcre Target)

1.  **Retainer Amount:** Value entered in form ($3000) is **not transferring** to Valcre (stays at $350 default).
2.  **Additional Notes/Comments Pollution:** Client's "Additional Notes" text is being incorrectly mapped to the Valcre field **Property $\rightarrow$ Off-Site Improvements** (Should go to **Job $\rightarrow$ Internal Comments** or a general comments field).

#### Formatting/Cleanup Issues

1.  **ClickUp Task Name:** Too long; needs to be shortened (e.g., remove **Province and Postal Code**).
2.  **Disbursement Field:** Field is **obsolete** (related to old employer workflow). Must be removed from the dashboard, database, and all mapping logic.

#### Field Mapping Cross-Reference

The core field structure across the Client Form, APR Dashboard, and ClickUp Custom Fields was analyzed, confirming excellent alignment in structure, with two critical data transfer bugs remaining.
