## 1. Executive Summary

- **Field count is severely stale.** Docs report **218 fields**. Production `fieldRegistry.ts` contains **2,082 fields** (matches root `CLAUDE.md`). That's a **~10x undercount** — the docs are documenting only the calculator subset (4 valuation approaches) and presenting it as the master total. Highest-impact drift in the folder.
- **All numbered docs have a "Last Updated: 2026-01-12" stamp** and a `Status: COMPLETE (Reference Only)` framing. That framing is wrong: an active inbox task exists ("VALTA Field Spec — 13 missing fields + 3 dropdown fixes"), so the registry is NOT frozen. The "Reference Only" label is misleading.
- **Valcre integration docs are 4.5 months stale** (Dec 29 2025) and pre-date the current `api/valcre.ts` (last touched Apr 1 2026, 1,540 lines). Six conversion maps exist in code; doc covers five (`SCOPE_OF_WORK_MAP` is missing from §3 enum conversions despite being referenced elsewhere in the doc).
- **Cross-links inside Valcre-Integration are broken.** `1-API-FIELD-MAPPING-REFERENCE.md` links to `07-Valcre-Integration/PROPERTY-TYPE-FIELD-MAPPING.md` 3 times — that subfolder does not exist at that path. The file lives in the SAME folder as the linking doc.
- **No `CLAUDE.md` in this folder.** Valta prototype has one; this folder doesn't. Given Ben's stated direction (Valta = client-facing, Master = deep reference), a companion `CLAUDE.md` is the cheapest single fix to close the loop and prevent agents from duplicating work.
