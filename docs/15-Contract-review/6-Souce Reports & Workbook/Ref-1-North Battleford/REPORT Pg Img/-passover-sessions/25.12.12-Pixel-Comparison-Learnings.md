# Pixel Comparison Approach - Learnings

**Date:** 2025-12-12
**Context:** PNG-to-HTML conversion QA workflow
**Session:** North Battleford Appraisal Report (79 pages)

---

## What We Tried

Used ImageMagick `compare -metric RMSE` to do pixel-perfect comparison between:
- Source PNG (exported from PDF)
- Rendered HTML screenshot (via Playwright)

**Goal:** Objective measurement of styling accuracy vs subjective LLM review.

---

## The Problem

**Pixel comparison treats page margin offset as content difference.**

### Example:
- **Source PNG:** May have 0.75" margins from PDF export
- **Our HTML:** Standardized 40px/50px padding
- **Result:** Entire page content appears "shifted" in pixel diff
- **Diff Score:** Inflated to 11.5% even if content layout is identical

### Visual Issue:
Diff overlay showed text "moved up" - but this was just margin offset, not actual content problems.

---

## Critical Insight

**We should NOT match arbitrary source PNG margins.**

### ❌ Wrong Approach:
- Match exact top/left/right page margins from PDF export
- Change HTML padding to align with source PNG borders
- Treat page-level spacing as a styling requirement

### ✅ Right Approach:
- Use **standardized page margins** across all HTML outputs
- Focus comparison on **content area** only (crop margins before diff)
- Review **content-specific styling**: tables, alignment, typography

---

## What Pixel Comparison IS Good For

1. **Content-area comparison** (after cropping page margins)
2. **Automated regression testing** for re-renders of same HTML
3. **Detecting unintended changes** when editing existing pages

---

## What Pixel Comparison is NOT Good For

1. ❌ Comparing pages with different margin settings
2. ❌ Detecting semantic styling issues (bold, alignment, small-caps)
3. ❌ Validating "close enough" matches (too binary: pixel-perfect or fail)

---

## Better QA Approach for This Project

### Manual Spot-Check (What We Did):
1. Visual side-by-side comparison
2. Identify specific styling issues:
   - Missing borders/lines in content
   - Column alignment (left/right/center)
   - Typography (small-caps, bold weights)
   - Content spacing (not page margins)
   - Table structure accuracy

### Automated Polish:
1. Sonnet agent reviews source PNG + HTML code
2. Fixes specific styling issues (not margins)
3. Focus: alignment, typography, content borders

---

## Recommendation for Future Sessions

**For new PDF-to-HTML conversions:**
- Use pixel diff on **cropped content area** only
- OR use manual spot-check + Sonnet polish for identified issues
- Do NOT try to match page-level margins from source PDF

**For HTML re-renders (regression testing):**
- Pixel diff works great (comparing HTML v1 → v2)
- Can detect unintended styling changes

---

## Action Taken

Skipped full-page pixel comparison.
Deployed Sonnet polish agent with 5 specific content styling fixes:
1. Add horizontal line above column headers
2. Right-align numeric column headers
3. Apply small-caps to section titles
4. Increase font-weight on table labels
5. Adjust content spacing (padding, line-height)

**Excludes:** Page margin/border adjustments.
