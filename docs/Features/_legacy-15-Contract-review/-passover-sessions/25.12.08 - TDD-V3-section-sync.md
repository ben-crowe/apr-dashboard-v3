### 2. V3 Operational Sections Added

Created two new sections at the top of TDD that mirror the deployed V3 Dashboard:

**Section 1: Client Intake (18 fields)**
- Client Information: First Name, Last Name, Email, Phone, Title, Organization, Address
- Property Information: Name, Address, Type, Intended Use, Valuation Premises, Asset Condition
- Property Contact: First Name, Last Name, Email, Phone
- Notes

**Section 2: LOE Prep (11 fields)**
- Job Assignment: Valcre Job ID (VAL#)
- Financial Terms: Appraisal Fee, Retainer Amount, Payment Terms
- Delivery Details: Delivery Date, Report Type, Property Rights Appraised
- Scope: Scope of Work, Special Instructions
- Comments: Internal Comments, Appraiser Comments

### 3. Field Mapping Architecture

Fields in S1/S2 have `mapsTo` property indicating where they auto-populate:

```typescript
{
  id: "intake-property-name",
  label: "Property Name",
  mapsTo: "property-name",  // Maps to cover section
}
```

**Mappings defined:**
- `intake-client-first-name` → `client-contact-name`
- `intake-client-organization` → `client-company`
- `intake-property-name` → `property-name`
- `intake-property-address` → `street-address`
- `intake-property-type` → `property-type-display`
- `intake-intended-use` → `assignment-intended-use`
- `intake-valuation-premises` → `value-scenario`
- `loe-valcre-job-id` → `file-number`
- `loe-report-type` → `report-type`
- `loe-property-rights` → `property-rights`
- `loe-scope-of-work` → `report-scope`

## Decisions Made

1. **Keep existing tabs unchanged** - Appraisers are familiar with Valcre workbook structure
2. **Add V3 tabs at top** - New S1/S2 sections for operational data entry
3. **Dual entry supported** - Fields exist in both S1/S2 and other tabs (for now)
4. **"Introduction Letter"** chosen over "Letter of Transmittal" for clarity
5. **Removed "custom" section** - Not needed in either app
6. **Consolidated VAL/Valcre ID** - They're the same thing (e.g., VAL251001)
