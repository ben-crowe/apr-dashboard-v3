### Data Flow Verification Best Practices

**What Worked:**
1. **Reference-based testing** - Extract actual values from PDF → trace through system
2. **Field type diversity** - Test different types (text, date, numeric, currency)
3. **Complete pipeline trace** - Registry → Store → Template → Output
4. **Don't skip formatting** - Verify $ and commas actually work (not just assume)
5. **Fix gaps immediately** - Don't accumulate technical debt (appraiser fields)

**Pattern Recognition:**
- Text/Date fields (Page 1) = basic rendering ✅
- Numeric/Currency fields (Page 6) = formatting layer ✅
- TEXTAREA/Calculations (Pages 12, 47) = advanced rendering (next)
- Images/Grids (Page 55) = layout complexity (next)
