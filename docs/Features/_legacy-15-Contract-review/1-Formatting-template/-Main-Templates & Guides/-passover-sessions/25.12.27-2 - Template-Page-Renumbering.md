## FOR HUMANS (What Happened)

**Primary Topic:** Aligned template page numbers with footer page numbers

**What We Built:**
- `Report-MF-template-LIVE-2025-12-27.html` → Updated to v2.4.0
- `APR-Template-Viewer.app` → Desktop launcher (localhost:8085)
- Updated both FIELD-IDS and FIELD-REFERENCE markdown files

**What We Discovered:**
- Template pages started at "Page 2" but should align with footer numbers
- Cover, Letter, Limiting Conditions, TOC have no footer → negative numbers
- Executive Summary is footer "Page 1" → should be Page 1 in data-page-num

**What Changed:**
- All `data-page-num` attributes: Page 2→-4, Page 3→-3, Page 4→-2, Page 5→-1, Page 6→1, etc.
- Version header updated with changelog entry
- Markdown files updated with descriptive page names

**Decisions Made:**
- **Decision:** Use negative numbers for non-footer pages
  - **Why:** Workaround for preview showing pages without footer page numbers
  - **Impact:** Pages -4 to -1 = no footer, Pages 1-67 = with footer

- **Decision:** Two-pass sed with markers
  - **Why:** Avoid cascading replacements (Page 7→Page 2→Page -4)
  - **Impact:** Clean renumbering without data corruption
