## Problems Discovered

### 1. Store Sections Empty on Init
- **Symptom:** `loadFullTestData()` reported "0/508 fields mapped"
- **Root Cause:** Store initialized with `sections: []` empty array
- **Location:** `reportBuilderStore.ts:5430`
- **Fix:** Call `initializeMockData()` in CalculatorDemoPage useEffect

### 2. Duplicate Test Data Entries
- **Symptom:** All calc field values were string 'calc' instead of numbers
- **Root Cause:** Test data file had duplicate calc field definitions at end
- **Location:** `northBattlefordTestData-REAL.ts:1071-1133`
- **Fix:** Removed duplicate entries that overwrote real values

### 3. Wrong Indicated Value ($3,280,000)
- **Symptom:** Calculator produced $3,280,000 instead of $1,780,000
- **Root Cause:** Parking/laundry values (375, 150) multiplied by units x 12
- **Formula:** `parkingTotal = parkingPerUnit * totalUnits * 12`
- **Fix:** Set parking/laundry to 0 (included in base rent)

### 4. $10K Rounding Difference
- **Symptom:** Got $1,790,000 instead of $1,780,000
- **Root Cause:** Rent $1,064/mo produces slightly high NOI
- **Fix:** Adjusted rent to $1,060/mo

## Technical Notes

### Store Initialization Pattern
The reportBuilderStore has sections defined in `getMockData()` function but initial state is `sections: []`. Any component using the store must either:
1. Call `initializeMockData()` first
2. Handle empty sections gracefully

### Zustand Re-render Pattern
When selecting from Zustand store, use individual selectors rather than destructuring:
```typescript
// Good - triggers re-render on sections change
const sections = useReportBuilderStore(state => state.sections);

// Bad - may not trigger re-render properly
const { sections } = useReportBuilderStore();
```

### Calculator Formula Reference
```
Rental Revenue = units × rent × 12
PGR = Rental + Other Income
Vacancy Loss = PGR × vacancy_rate
EGR = PGR - Vacancy Loss
Expenses = management% × EGR + per_unit_expenses × units
NOI = EGR - Expenses
Value = NOI / cap_rate
Indicated = round(Value / 10000) × 10000
```
