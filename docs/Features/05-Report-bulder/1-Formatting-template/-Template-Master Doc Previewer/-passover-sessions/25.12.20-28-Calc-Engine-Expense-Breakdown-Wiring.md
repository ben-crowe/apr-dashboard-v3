## 📝 TECHNICAL NOTES

### Field Output Pattern

For each expense category (taxes, insurance, repairs, payroll, utilities, management, other):

| Metric | Field ID | Type | Formula |
|--------|----------|------|---------|
| Annual | calc-exp-{name}-annual | currency | value (rounded) |
| Per-Unit | calc-exp-{name}-per-unit | currency | value / totalUnits |
| Per-SF | calc-exp-{name}-per-sf | currency | value / totalSf |
| % of PGR | calc-exp-{name}-pct-pgr | percentage | (value / pgr) × 100 |
| % of EGR | calc-exp-{name}-pct-egr | percentage | (value / egr) × 100 |

### Calc Base System
- Expenses use different calculation bases:
  - `per_unit`: value × total units (most expenses)
  - `percent_egr`: EGR × percentage (management)
  - `percent_pgr`: PGR × percentage (if used)
  - `fixed`: fixed amount
  - `per_sf`: value × total SF

The calculateExpense() function handles base selection automatically.

### Performance Impact
- Minimal: forEach loop runs once per calculation
- Loop adds ~0.5ms overhead (negligible)
- Most time spent in getFieldValue() calls (already optimized)
