## Decision: Migration Approach

**Pattern**: Same as `image-mgt` (S3) - single source of truth

### How It Will Work

| Source Section | Feeds Into | Method |
|----------------|------------|--------|
| `client-intake` (S1) | Cover, Report fields | `getGlobalFieldValue()` |
| `loe-prep` (S2) | Exec, Report fields | `getGlobalFieldValue()` |
| `image-mgt` (S3) | Cover, Maps, Photos | `getGlobalFieldValue()` *(already done)* |

### What Will Change (NOT YET IMPLEMENTED)

1. **Template** (`reportHtmlTemplate.ts`):
   - Read client data from S1 fields using `getGlobalFieldValue('intake-*')`
   - Read LOE data from S2 fields using `getGlobalFieldValue('loe-*')`

2. **"Load Test Data" button**:
   - Stop populating Cover/Exec client fields directly
   - Populate S1/S2 fields instead
   - Template reads from S1/S2 → displays in Cover/Exec sections

3. **TDD Tabs** (Cover, Exec, etc.):
   - Fields remain visible in their current tabs
   - Data flows FROM S1/S2 TO display locations
