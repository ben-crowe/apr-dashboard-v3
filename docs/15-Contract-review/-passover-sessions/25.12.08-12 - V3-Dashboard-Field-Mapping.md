# V3 Dashboard Operational Fields - TDD Mapping Reference

**Last Updated:** December 8, 2025  
**Purpose:** Identify which fields in TDD (Test Data Dashboard) correspond to operational V3 Dashboard fields already working in production

---

## 🔄 Data Flow Architecture

```
Client Form Submission (Intro Letter)
          ↓
V3 Dashboard Section 1 & 2 (OPERATIONAL)
          ↓
VAL Job Number Created (Valcre API)
          ↓
TDD Dashboard (Test Mirror - Pre-Integration)
          ↓
Report Builder (Final Output)
```

**Key Points:**
- Section 1 & 2 fields in V3 Dashboard are **already operational** in production
- TDD mirrors these fields for testing before full Report Builder integration
- Once tested, TDD data will be replaced with direct V3 Dashboard → Report Builder connection

## 🔗 Field Sync Logic

### Auto-Sync Fields (Client Form → V3 Dashboard)
These fields automatically sync when client submits the form:
- All Client Information fields (7 fields)
- All Property Information fields (6 fields)
- All Property Contact fields (4 fields)
- Client Comments (1 field)
- Uploaded Documents (files array)

### Manual Entry Fields (Staff → V3 Dashboard)
These fields are filled by staff during property research:
- All Property Site fields (5 fields)
- All Parcels Summary fields (8 fields)
- All Assessments & Taxes fields (6 fields)

### API-Generated Fields (Valcre → V3 Dashboard)
These fields are auto-created via Valcre integration:
- VAL Job Number
- Valcre Job ID

- **Property Types:** Multi-select field stored as comma-separated string
- **Phone Formatting:** Auto-formats to (XXX) XXX-XXXX
- **Currency Formatting:** Auto-formats with $ and commas
- **Auto-Save:** All fields auto-save on blur with 500ms debounce
