## 🏗️ THE CALCULATOR DEMO PAGE DESIGN

### Vision

Build an interactive demonstration page where stakeholders can:
1. Load validated test data (North Battleford) with one button click
2. See or modify all 62 calculator input fields
3. View 7 key results updating in real-time
4. **See step-by-step calculation breakdown** showing HOW each result was derived

**User's Insight:** *"Wouldn't it be cool after reading this validation report my next thing would be okay, let me play with it"*

### 3-Panel Architecture

```
┌─────────────────────────────────────────────────────────┐
│  Header: Calculator Demo | Load Test Data | Reset       │
├──────────────────┬──────────────────┬───────────────────┤
│  INPUT PANEL     │  OUTPUT PANEL    │  WALKTHROUGH      │
│  (Left/Top)      │  (Right/Top)     │  (Bottom/Full)    │
│                  │                  │                   │
│  62 Fields:      │  7 Results:      │  8 Step Groups:   │
│  • Unit Mix (20) │  • PGR           │  1. Unit Mix      │
│  • Totals (6)    │  • Vacancy Loss  │  2. Other Income  │
│  • Other Inc (6) │  • EGR           │  3. PGR           │
│  • Vacancy (5)   │  • OpEx          │  4. Vacancy/Loss  │
│  • Expenses (11) │  • NOI           │  5. Expenses      │
│  • Cap Rate (1)  │  • Raw Value     │  6. NOI           │
│  • Adjust (4)    │  • Final Value   │  7. Cap Value     │
│                  │                  │  8. Adjustments   │
└──────────────────┴──────────────────┴───────────────────┘
```

### Key Innovation: The Walkthrough Panel

**Problem:** Stakeholders see $1,780,000 value but don't understand HOW it was calculated

**Solution:** Show step-by-step breakdown with formulas

**Example Display:**
```
Step 3: Potential Gross Revenue (PGR)

Formula: PGR = Rental Revenue + Other Income
Calculation: PGR = $204,288 + $8,400 = $212,688

Inputs Used:
  • Rental Revenue: $204,288 (from Unit Mix calculation)
  • Other Income: $8,400 (Parking + Laundry)

Result: $212,688
```

**All 8 steps shown:**
1. Unit Mix → Total Rental Revenue
2. Other Income → Total Other Income
3. PGR = Rental + Other
4. Vacancy & Loss → EGR
5. Operating Expenses → Total OpEx
6. NOI = EGR - OpEx
7. Capitalized Value = NOI / Cap Rate
8. Final Adjustments → Indicated Value

**Benefits:**
- Transparency builds trust
- Educational for clients
- Demonstrates methodology rigor
- Validates calculator is working correctly
- Can be used for presentations

## 🔧 TECHNICAL APPROACH

### Critical Constraint: DO NOT MODIFY CALCULATOR ENGINE

**Calculator Engine Location:**
- File: `/src/features/report-builder/store/reportBuilderStore.ts`
- Function: `runCalculations()` (lines 5534-5727)
- Status: ✅ VALIDATED - 7/7 exact match with Valcre

### Non-Invasive Wrapper Solution

**Problem:** Calculator engine doesn't expose intermediate calculation steps

**Solution:** Create parallel step-tracking logic

**Implementation:**
```typescript
// useCalculationSteps.ts hook
export function useCalculationSteps(): CalculationStep[] {
  const store = useReportBuilderStore();

  // Subscribe to all relevant store fields
  // Re-implement same calculation logic to capture steps
  // Return formatted step objects with formulas

  const steps: CalculationStep[] = [];

  // Example: Unit Mix Step
  const type1Count = getFieldValue('calc-type1-count');
  const type1Rent = getFieldValue('calc-type1-rent');
  const type1Annual = type1Count * type1Rent * 12;

  steps.push({
    id: 'unit-mix-type1',
    category: 'Unit Mix',
    label: 'Type 1 Annual Revenue',
    formula: '{count} units × ${rent}/mo × 12',
    inputs: [
      { name: 'count', value: type1Count, unit: 'units' },
      { name: 'rent', value: type1Rent, unit: '$/month' }
    ],
    result: type1Annual,
    resultFormatted: formatCurrency(type1Annual)
  });

  // ... continue for all 8 step categories

  return steps;
}
```

**Advantages:**
- Does NOT modify validated calculator
- Keeps engine pristine and validated
- All step-tracking logic isolated
- Easy to test independently
- No risk to production calculator

**Trade-off:**
- Duplicates some calculation logic
- Requires manual sync if calculator changes
- Acceptable because calculator is stable and validated
