## Topics Covered

### 1. Image Field Architecture
- **Single source of truth**: All images stored in `image-mgt` section (S3 IMAGE MGT in TDD)
- **Template lookup**: Uses `getGlobalFieldValue()` to find images regardless of which section they're stored in
- **EditPanel display**: Related images now appear inline in Cover, Maps, Photos sections

### 2. Cover Photo Fix
- **Problem**: Cover photo wasn't loading in preview
- **Root cause**: Template was looking for `img-cover-photo` only in `cover` section, but field is stored in `image-mgt` section
- **Solution**: Added `getGlobalFieldValue()` helper that searches ALL sections

### 3. EditPanel Related Images
- **Feature**: When viewing Cover/Maps/Photos sections, related images from `image-mgt` appear at bottom
- **UI**: Green "SECTION IMAGES" header with grid layout
- **Mapping**: `SECTION_IMAGE_MAPPING` constant defines which images belong to which sections

### 4. TDD Reload Button
- **Feature**: Added "Reload" button to TDD header for quick page refresh
- **Purpose**: Helps when HMR glitches occur without needing terminal access

---

## Decisions Made

| Decision | Rationale |
|----------|-----------|
| Keep images in `image-mgt` only | Single source of truth, no duplication |
| Use global field lookup | Images may be stored in different section than where they're displayed |
| Show images in EditPanel | User can edit images inline without switching tabs |
| Shared Zustand store | Enables bidirectional sync between TDD and Report Builder |

## Architecture Notes

### Image Flow
```
S3 IMAGE MGT (TDD)
    ↓ stores to
Zustand Store (image-mgt section)
    ↓ read by
EditPanel (shows in Cover/Maps/Photos)
    ↓ read by
reportHtmlTemplate (getGlobalFieldValue)
    ↓ renders to
Preview / PDF Export
```
