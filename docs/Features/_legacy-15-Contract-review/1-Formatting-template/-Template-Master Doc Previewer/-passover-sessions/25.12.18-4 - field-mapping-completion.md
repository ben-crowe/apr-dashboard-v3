## Key Discoveries

### Discovery 1: Template Architecture Limitation
**Problem:** Pages 37-40 (Rental Survey) require dynamic/repeating rows based on variable unit counts.

**Current Template:** Fixed field placeholders (`{{comp1-name}}`, `{{comp2-name}}`, etc.)
- ✅ Works for fixed tables: Page 49 (80 fields), Page 59 (188 fields), Page 61 (68 fields), Page 63 (14 fields)
- ❌ Doesn't work for variable rows: Page 40 adjustment grid (5-20 rows per unit type)

**Solution Needed:** Add loop/iteration support to template engine
- Options: Handlebars `{{#each}}`, Mustache, custom JSX `.map()`, or built custom processor
- Impact: Would allow array-based data instead of flat fields

### Discovery 2: Calculation Engine Gap
**Current Status:**
| Approach | Calc Engine | Status |
|----------|-------------|--------|
| Income (Direct Cap) | ✅ Built | Validated to $1,780,000 |
| Sales Comparison | ❌ Not built | Simple (division/sums) |
| Rental Survey | ❌ Not built | **Complex** (statistical + aggregation) |

**Why Rental Survey is Complex:**
- Requires HIGH/AVG/MED/LOW calculations (not just simple division)
- Requires unit type splitting (1-bed, 2-bed, 3-bed arrays)
- Requires adjustment matrix application before statistical calcs
- Current workaround: Manual rent input to Income Approach works fine

### Discovery 3: Data Structure Redesign Needed
**Current:** Flat fields for fixed tables
```typescript
{ 'rental-comp1-name': '...', 'rental-comp1-address': '...', 'rental-comp2-name': '...', ... }
```

**Required for Pages 37-40:** Array-based data
```typescript
{
  rentalComps1Bed: [
    { compNum: 1, name: '...', address: '...', unitSize: 650, rentUnit: 1575, ... },
    { compNum: 2, name: '...', address: '...', unitSize: 650, rentUnit: 1110, ... },
    // ... variable length
  ]
}
```

## Architectural Insights for Future Work

### Template Engine Gap
The report builder uses fixed field placeholders which work perfectly for:
- Tables with known structure and row count
- Examples: Pages 49 (80 fields), 59 (188 fields), 61 (68 fields), 63 (14 fields)

But fails for:
- Tables with variable row counts
- Nested arrays or repeating sections
- Examples: Pages 37-40 (rental survey with 5-20 unit types per comp)

**Solution:** Template engine must support loops/iteration. Research needed:
- Does `generateReportHtml()` already support loops?
- If not: Add Handlebars/Mustache library OR build custom loop processor
- Impact: Would allow array-based data flow instead of flat field mapping

### Calculation Engine Priorities
1. **Sales Comparison Calc:** Low complexity, high value
   - Simple: `pricePerUnit = salePrice / units`
   - Statistical: HIGH/AVG/MED/LOW across 5 comps
   - Would enable Page 59 → Page 61 automation

2. **Rental Survey Calc:** High complexity, deferred
   - Statistical: HIGH/AVG/MED/LOW by unit type
   - Adjustment: Apply quality/condition/location adjustments first
   - Would enable Pages 37-40 automation + feed into Income Approach

### Data Structure Evolution
- **Phase 1 (Current):** Flat fields work for fixed tables
- **Phase 2 (Next):** Add template loops for variable tables
- **Phase 3 (Future):** Migrate to array-based data structures for complex aggregations
