**What Changed:**
- **fieldRegistry.ts:** Applied sed patterns for kebab-case (sale-price, cap-rate, year-built, etc.)
- **northBattlefordTestData.ts:** Applied same kebab-case conversions
- **Dev server:** Restarted on port 8082 (from 8084) and cleared Vite cache
- **Understanding:** Confirmed ALL sections need dynamic building, not just image-mgt

**Decisions Made:**
- **Decision:** Build all sections dynamically from fieldRegistry, not just image-mgt
  - **Why:** Current hardcoded approach only includes 73 fields out of 1,687 in registry
  - **Impact:** Will allow all 1,101 test data fields to map successfully to store

- **Decision:** Kebab-case conversions were unnecessary but applied
  - **Why:** Fields were already in correct format (subject-name, comp1-sale-price, etc.)
  - **Impact:** No negative effect, files already had matching names

- **Decision:** Use buildImageMgtSection() pattern for all sections
  - **Why:** It successfully builds sections from fieldRegistry by filtering on section/subsection
  - **Impact:** Can iterate through all unique section/subsection combinations in registry
