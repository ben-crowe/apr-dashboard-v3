## 🔧 Decisions Made

| Decision | Rationale |
|----------|-----------|
| **Copy workbookFieldMapping.ts to schema folder** | Single source of truth for Valcre mapping |
| **Use kebab-case id === storeId** | Matches existing pattern, avoids conversion layers |
| **Contract rent = user-input** | User provides actual lease rates |
| **Derived metrics = calculated** | System derives from annual amounts |
| **Add -annual suffix** | Naming consistency with template (Tax_Annual) |
| **All calc section** | Keeps calculation engine separate from property data |
| **Separate subsections** | calc-unit-mix, calc-expenses organize by category |
| **7 expense categories** | Matches HTML template structure (Taxes, Insurance, Repairs, Payroll, Utilities, Management, Other) |

## 🐛 Problems Solved

### Problem 1: Expense Breakdown Fields Missing
**Root Cause:** Page 49 template uses 5 columns per expense (pct-pgr, pct-egr, per-unit, per-sf, annual) but registry only had annual amount

**Solution:** Added 35 fields (7 categories × 5 columns) with correct inputSource classification:
- Percentages: calculated (formula-driven)
- Per-unit/per-sf: calculated (derived from annual ÷ units/sf)
- Annual: user-input (source value)

**Result:** ✅ All 70 expense table cells now mapped

### Problem 2: Unit Mix Contract Rent Fields Missing
**Root Cause:** Registry had market rent but not contract rent for comparison

**Solution:** Added 16 fields:
- calc-type{1-4}-contract-rent (user-input)
- calc-type{1-4}-cont-v-market (calculated, percentage)
- calc-type{1-4}-per-unit (calculated, currency)
- calc-type{1-4}-per-sf (calculated, number)

**Result:** ✅ Unit mix section complete with dual rent analysis

### Problem 3: Table Page Locations Unclear
**Root Cause:** Multiple tables exist, unclear which page each occupies

**Solution:** Systematically mapped PREVIEW-Master.html to identify:
- Pages 54-58: Individual comp detail sheets
- Page 59: Sales Comparison Grid (adjustment grid)
- Page 60: Analysis narrative
- Page 61: DCA Conclusion
- Page 62-63: Reconciliation

**Result:** ✅ All 23 tables anchored to specific pages

### Problem 4: Field Naming Consistency
**Root Cause:** Some fields lacked -annual suffix (calc-exp-taxes vs Tax_Annual)

**Solution:** Added -annual suffixed versions for template mapping clarity:
- `calc-exp-taxes-annual` vs existing `calc-exp-taxes`
- Allows explicit 1:1 mapping to template fields
- No functional duplication - can be aliased if needed

**Result:** ✅ 51 new fields follow clear naming pattern

## 📝 Lessons Learned

1. **Pattern Recognition:** Consistent field naming (kebab-case, id === storeId) enables predictable mappings
2. **Breakdown Strategy:** Separating user-input from calculated fields creates clear data flow
3. **Template-First:** HTML template analysis reveals exact field requirements before registry work
4. **Audit First:** Pre-audit prevents duplicate work and validates patterns
5. **Batch Approach:** Adding fields in logical groups (Valcre, Page 49) maintains atomic commits
