## 🎓 Key Learnings

### For Future Sessions

1. **PREVIEW-Master.html is the single source of truth** - don't generate, just populate
2. **Zoom is essential** - users can't work on full-page apps without it
3. **Toggle for field IDs is powerful** - shows {{field-id}} in test mode, hides in production
4. **Calc engine is reliable** - focus on output wiring, not math fixes
5. **Three-agent structure works** - specialist agents with clear responsibilities faster than generalist

### Patterns for Reuse

- Template loading + field interpolation pattern works for any field-based system
- Zustand store + React components sharing state = clean data flow
- Field registry as single source of truth prevents mismatch bugs
- {{placeholder}} format is simple but powerful for template systems
