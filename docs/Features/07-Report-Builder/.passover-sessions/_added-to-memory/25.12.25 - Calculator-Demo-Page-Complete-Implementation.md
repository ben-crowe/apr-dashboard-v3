## Executive Summary

The Calculator Demo Page is a fully functional Income Capitalization (Direct Capitalization) valuation calculator that demonstrates real-world property valuation capabilities. The app has been updated from hardcoded test numbers to accept **human input data** across all three valuation approaches (Income, Sales Comparison, Cost) with full reconciliation reporting.

## Data Flow & Reactivity

### State Management
**Store:** `useReportBuilderStore` (Zustand)

All data flows through Zustand store:
1. **User inputs** → `updateFieldValue()`
2. **Store updates** → triggers `runCalculations()`
3. **Calculations** → all fields updated in store
4. **Components subscribe** → re-render with new values

### Key Functions:
- `updateFieldValue(fieldId, value)` - Update single field
- `runCalculations()` - Recalculate all derived fields
- `loadFullTestData()` - Load North Battleford dataset
- `initializeMockData()` - Initialize store on mount

### Auto-Calculated Fields:
Revenue calculations:
- Total Units = sum of unit types
- Total SF = Total Units × Unit SF
- Total Annual Revenue = sum of unit revenues
- PGR = Rental Revenue + Other Income
- EGR = PGR - Vacancy Loss

Expense calculations:
- All expenses: input × units or % × EGR
- Total Expenses = sum of all expense line items
- Expense Ratio = Total Expenses ÷ EGR

Value calculations:
- NOI = EGR - Total Expenses
- Raw Value = NOI ÷ Cap Rate
- Indicated Value = Rounded Raw Value

## Data Entry Workflow (User Perspective)

### Scenario 1: Load Test Data
1. Click "Load North Battleford Test Data" button
2. All fields populate with example values
3. Income/Sales/Cost calculations auto-generate
4. Adjust any inputs as needed
5. Reconciliation section combines all three approaches
6. Final value displayed at bottom

### Scenario 2: Enter Custom Property Data
1. Enter Unit Mix (count, SF, rent)
2. Add Other Income (parking, laundry)
3. Set Vacancy/Loss rates
4. Enter Operating Expenses (as % or per unit)
5. Set Cap Rate
6. Expand Income Approach to see detailed breakdown
7. Enter comparable sales data (Sales Comparison section)
8. Enter cost-based data (Cost Approach section)
9. Adjust reconciliation weights
10. Final value reconciles all three approaches
